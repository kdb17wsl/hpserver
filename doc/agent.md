# **企业级正向代理服务器项目概述（AI Agent专用版）**  


## **一、项目定位**  
**名称**：企业级HTTP/HTTPS正向代理服务器（C++17/20实现）  
**核心角色**：内网设备访问外网的“安全网关”，解决企业内网**安全管控、效率优化、合规审计**三大痛点。  
**价值主张**：通过高并发、低延迟的网络中转能力，实现“可控访问、资源复用、行为可追溯”，替代传统冗余代理工具（如Squid），提供轻量化、可定制的代理服务。  


## **二、核心目标**  
### **1. 必做目标（核心能力）**  
| **维度**       | **目标描述**                                                                 | **量化指标**                          |  
|----------------|-----------------------------------------------------------------------------|---------------------------------------|  
| **协议支持**   | 支持HTTP/1.1（`keep-alive`/`chunked`）、HTTPS（`CONNECT`隧道）解析与转发       | 解析准确率≥99%，HTTPS隧道建立成功率≥99% |  
| **高并发**     | 基于epoll Reactor+多线程模型，支撑内网数千设备并发访问                         | 单节点并发5000+，QPS≥8000，平均延迟<50ms |  
| **访问控制**   | IP白名单（CIDR格式）+ URL正则过滤，动态配置热更新（SIGHUP信号）                 | 日均拦截非法请求≥200次，配置更新耗时<1s |  
| **缓存优化**   | 1GB LRU缓存（哈希表+双向链表），按`Cache-Control`管理过期，支持`304 Not Modified` | 缓存命中率≥45%，减少外网带宽占用≥30%    |  
| **日志审计**   | 异步日志（JSON格式，生产者-消费者模型）+ 关键指标（QPS/命中率/拦截次数）       | 日志写入延迟<1ms，支持按日生成审计报告  |

## **三、架构设计**  
### **1. 整体架构**  
**分层模块化+事件驱动**，6大核心模块协作：  
```  
客户端请求 → 网络层（epoll Reactor+线程池）→ 协议解析（HTTP/HTTPS/SOCKS5）→ 访问控制（IP/URL校验）→ 缓存检查（LRU）→ 请求转发（目标连接池）→ 响应回传（更新缓存）→ 日志审计（异步记录）  
```  


### **2. 核心模块**  
| **模块**           | **职责**                                                                 | **关键技术**                                                                 |  
|--------------------|-------------------------------------------------------------------------|-----------------------------------------------------------------------------|  
| **网络层**         | 高并发接入（epoll ET模式）、双端连接池（客户端/目标服务器）、线程池任务调度   | epoll事件驱动、非阻塞IO、引用计数防泄漏、多线程Reactor模型                   |  
| **协议解析**       | 解析HTTP/1.1（请求行/首部/Body）、HTTPS（`CONNECT`隧道）、SOCKS5（RFC 1928） | 粘包/拆包处理（`Content-Length`/`chunked`）、隧道透传加密数据                 |  
| **访问控制**       | IP白名单（CIDR校验）、URL正则过滤（`.exe$/phishing\.com`）、动态配置热更新   | `std::regex`匹配、INI配置解析（`inih`库）、SIGHUP信号重载                     |  
| **缓存模块**       | LRU缓存（1GB容量）、过期策略（`max-age`）、条件请求（`If-Modified-Since`）   | 哈希表+双向链表、线程安全（`std::mutex`）、缓存键（URL+方法+版本）             |  
| **请求转发**       | 目标服务器连接复用、数据透传（含`sendfile`零拷贝）、错误处理（超时/熔断）     | 连接池管理、非阻塞IO写缓冲、`SO_RCVTIMEO/SO_SNDTIMEO`超时控制                 |  
| **日志审计**       | 异步日志（JSON格式）、指标统计（QPS/命中率）、对接ELK/Prometheus           | 生产者-消费者模型、批量刷盘、`trace_id`全链路追踪                             |  


## **四、关键技术亮点**  
### **1. 网络编程**  
- **epoll Reactor模型**：边缘触发（ET）模式+O(1)事件查询，支撑5000+并发连接；  
- **双端连接池**：客户端/目标服务器连接复用，减少TCP三次握手开销；  
- **线程池**：主Reactor分发事件，工作线程并行处理协议解析/缓存检查（默认8线程，按CPU核心数动态调整）。  


### **2. 协议与数据处理**  
- **HTTP/HTTPS解析**：支持`keep-alive`长连接、`chunked`分块传输、`gzip`压缩；HTTPS通过`CONNECT`建立隧道，透传加密数据；  
- **SOCKS5扩展**：实现握手（`VER=0x05`）、`CONNECT`命令，支持任意TCP协议转发（如SSH/FTP）；  
- **零拷贝优化**：用`sendfile`系统调用转发静态资源（避免用户态/内核态数据拷贝）。  


### **3. 安全与效率**  
- **访问控制**：IP白名单（CIDR网段）、URL正则过滤（拦截恶意站点）、动态配置热更新；  
- **缓存策略**：LRU算法+过期控制（`Cache-Control`）+条件请求（`304 Not Modified`）；  
- **安全增强**：WAF规则（SQL注入/XSS拦截）、令牌桶流量限速（单IP 10MB/s）、双向TLS加密（代理-目标服务器）。  


### **4. 工程化与可观测性**  
- **容器化**：Docker镜像（Alpine基础，体积小）+ Docker Compose编排（含Prometheus/Grafana）；  
- **CI/CD**：GitHub Actions自动化（编译→单元测试→压测→镜像推送）；  
- **监控**：Prometheus指标（QPS/缓存命中率/连接数）+ Grafana仪表盘+ ELK日志分析。
